# UE Mod Manager 故障排查

> 出问题先翻这本。每条都按"现象 → 原因 → 解决"三步走。
> 实在搞不定，**导出诊断包**（管理中心 → 导出诊断包）发开发者邮箱：mr.xzuo@foxmail.com

---

## 启动相关

### 双击图标没反应 / 闪退

| 排查 | 怎么做 |
|------|-------|
| 缺 .NET 8 运行时 | 装 [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/8.0)（安装包内置过，但精简版系统可能缺）|
| 杀毒软件拦截 | 把安装目录加白名单，重新启动 |
| 上次崩溃残留 | 去 `%APPDATA%\UEModManager\` 把 `local.db-shm` `local.db-wal` 删掉再开 |
| 看日志 | 软件目录的 `console.log`，最后几行通常有线索 |

### 启动后白屏 / 转圈圈卡死

- 等 30 秒（首次启动会做健康检查 + 扫描事务）
- 还卡 → 任务管理器结束 `UEModManager.exe`，再试一次
- 持续卡 → 删 `%APPDATA%\UEModManager\local.db*` 三个文件（注意先备份），重启

## 登录 / 账号

### 收不到验证码 / 找回密码邮件

- 检查垃圾邮件箱
- QQ 邮箱 / 163 邮箱有时延迟，等 5-10 分钟
- 仍无 → 用其他邮箱试试，或选"跳过"用离线模式
- 服务真挂了 → 联系开发者邮箱

### 登录提示"网络错误"

- 公司/学校网络可能屏蔽 → 切手机热点试试
- 完全不想联网 → 登录窗口选"跳过"，离线模式照常用

### 登录后头像不显示 / 显示离线

- 头像点一下 → 重新登录
- 还不行 → 设置里"账号"重新填密码

## 导入 MOD

### 拖文件进窗口没反应

- 看文件类型是不是支持的：`.zip` `.rar` `.7z` `.pak` `.utoc` `.ucas` `.dll` `.ini` `.json` `.cfg`
- 文件**正在被压缩软件打开** → 关掉压缩软件再拖
- 点"导入"按钮，从弹出的窗口选文件试试

### 压缩包导入很慢 / 卡死

- 压缩包太大或嵌套层数深，耐心等
- 真的卡死 → 解压到一个文件夹，把里面的 `.pak` 等文件单独拖进来

### 导入后看不到 MOD

- 检查左侧分类筛选，可能选错分类了 → 点"全部"
- 搜索框是否有文字 → 清空
- 重启程序

## 启用 / 禁用 MOD

### 开关打开后游戏里没效果

| 排查 | 怎么做 |
|------|-------|
| 游戏路径错了 | 设置 → 路径，重新选游戏根目录（`xxx.exe` 那一层）|
| 被其他 MOD 覆盖 | "冲突分析"看一下，手动选一个 MOD 赢 |
| 优先级太低 | 主列表里把这个 MOD 拖到上面 |
| MOD 本身和游戏版本不兼容 | 找 MOD 作者要新版 |

### 关闭开关后游戏目录还有残留文件

- v2.0-rc2 已修复此问题，会自动还原
- 如果是从老版本升级上来的旧数据 → 把开关**重新打开再关掉**触发一次修复
- 仍有残留 → 管理中心 → 仓库 → 看下游戏目录，必要时手动删

### 启用过程中卡住 / 报错

- **不要强关程序**，等它自己结束
- 真的卡死 → 任务管理器结束程序，重新启动**会自动弹"恢复"窗口**（参见使用说明 §10）
- 报错信息里有"权限"字样 → 用管理员身份运行管理器再试

### 删 MOD 提示"被方案引用，无法删除"

这是**安全保护**，不是 bug。

正确流程：

1. 切到使用这个 MOD 的方案
2. 把这个 MOD 的开关关掉
3. 再回来删

**或者**点强制删除（如果按钮可见）—— 但建议先关开关，更安全。

## 部署 / 冲突

### 提示"磁盘空间不足"

- 看 MOD 部署目录所在盘是否快满了
- 临时方案：在设置里把 MOD 部署目录改到大盘
- **硬链接 / 符号链接**模式比 复制 模式省空间，但要求 MOD 和游戏同盘

### 冲突分析里看到很多冲突

- 红色 = 严重，必须处理
- 黄色 = 警告，看情况
- 灰色 = 信息，不影响
- 处理：点冲突 → 选一个 MOD 作为"赢家"

## 启动游戏

### 点"启动"游戏没开

- 看"游戏路径"设的对不对，必须是含 `xxx.exe` 那层
- 游戏可能正在更新 / 启动器锁定了 exe → 先用启动器开一次再回管理器

### 游戏开了但 MOD 没生效

- 先确认主界面里 MOD 开关是**打开的**（绿色）
- 部署方式如果是"硬链接"或"符号链接"，确认 MOD 文件和游戏在同盘
- 用部署预览看一眼程序到底把文件写到哪儿了，对比游戏目录

## 数据迁移（老版本升级）

### 迁移向导一直转圈

- 数据量大时正常，最多等 5 分钟
- 真卡死 → 强关程序，**老数据已经备份**（如果你用了 `一键迁移老版本数据.bat`），把备份目录的内容覆盖回 `%APPDATA%\UEModManager\`，再试一次
- 仍失败 → 联系开发者，附 `console.log`

### 迁移后旧 MOD 不见了

- 迁移**不会**删旧文件，先去 `%APPDATA%\UEModManager-backup-xxx\` 看一眼是不是还在
- 在管理中心 → "数据迁移"可重新触发
- 实在不行就把备份目录的内容覆盖回去重新装

### 不知道老数据在哪

- 资源管理器地址栏粘贴 `%APPDATA%` 回车
- 找 `UEModManager` 文件夹

## 多游戏数据混乱

**症状：** 切游戏后看到上一个游戏的 MOD / 启用状态。

**原因：** v2.0-rc2 已知问题（S1/S2），多游戏当前共享一个数据库。下个版本会修。

**临时规避：**

- 每个游戏建独立的"方案"，切游戏时也切方案
- 出现错乱时重启程序

## 数据备份与还原

### 怎么备份我的全部配置？

复制以下两个目录：

1. `%APPDATA%\UEModManager\`（账号、分类、设置）
2. 软件安装目录下的 `Data\` 文件夹（MOD 包仓库、备份）

存到 U 盘 / 网盘即可。

### 怎么还原？

1. 装好相同版本的管理器
2. 把上面两个目录贴回原位置覆盖
3. 启动程序

### 一键迁移脚本备份的东西放哪了？

`%APPDATA%\UEModManager-backup-时间戳\`

资源管理器地址栏粘贴 `%APPDATA%` 回车就能看到。**确认迁移成功一周以上**可以手动删了释放空间。

## 紧急情况：把游戏目录恢复到出厂状态

如果某次操作后游戏怎么都不正常，想把游戏目录还原到没装 MOD 的状态：

1. 主界面把所有 MOD 的开关都**关掉**
2. 等程序处理完
3. 检查游戏目录是否干净
4. **不行的话**：用游戏启动器（Steam / Epic / 网易等）的"验证文件完整性"功能，把游戏文件从云端重新下回来

**最坏情况备份：** 软件目录 `Data\Backups\` 下每次部署的事务都有备份，可以手动从那里找原文件。
