## 30 秒理解

一个 .NET 8.0 多项目解决方案，管理虚幻引擎游戏 MOD，支持 9 款游戏。**5 个项目协同：**

| 项目 | 职责 | TFM | 依赖 |
|------|------|-----|------|
| `UEModManager/` | WPF 主程序（IO / UI / Service 编排） | net8.0-windows | EF Core / Supabase / CUE4Parse 等完整栈 |
| `UEModManager.Core/` | Domain 内核（模型 / 纯函数 / 算法 / 决策） | **net8.0** | **仅 Newtonsoft.Json** |
| `UEModManager.Core.Tests/` | xUnit 单元测试 | net8.0-windows | xUnit |
| `samples/UEModManager.SampleAdapter/` | 第三方 Host Adapter SDK 示例 | net8.0 | 仅 Core |
| `samples/UEModManager.SampleBackend/` | 第三方 Deployment Backend SDK 示例 | net8.0 | 仅 Core |

**核心模式：主项目 = "干活的"（IO / UI / Service 编排），Core = "想清楚的"（数据 / 算法 / 决策）。**

## v2.0-rc2 关键变化（修复批 A，2026-04-30）

经全面代码库审查（数据隔离 / 启用·禁用·卸载状态机 / 事务回滚），识别 14 项风险，本批完成 7 项严重 + 重要修复：

### 🛡 事务安全网

| 修复点 | 之前问题 | 修复方式 |
|--------|---------|---------|
| **S5** | `RollbackAsync` 单步失败 catch 吞异常，最终仍标 `RolledBack`，扫描器永远扫不到 | 加 `DeploymentStatus.PartiallyRolledBack` + `RollbackFailure` 列表，失败累积、状态如实标记 |
| **S6** | `ExecuteAsync` 在 `finally` 才写 `transaction.json`，写盘前崩溃事务消失 | 创建事务后立即写盘一次（`InProgress` 可见） |
| **S7** | `RollbackActionPlanner` 在备份缺失时返回 `None`，回滚静默"成功" | 加 `BackupMissing` / `NoBackupRecorded` 类型，注入 `backupExists` Func 让纯函数感知 IO；冒泡为 `PartiallyRolledBack` |
| **W5** | `SaveTransactionLogAsync` 写盘失败 catch 吞异常 + Status 已是 Committed | 写盘失败时降级为 `LogPersistenceFailed` |
| **W6** | 用户取消恢复对话框 → 事务保持 InProgress → 下次启动循环弹窗 | 加 `Dismissed` 状态 + `DismissTransactionAsync` / `ResetDismissedAsync` API |

### 🛡 状态机收紧

| 修复点 | 修复方式 |
|--------|---------|
| **S3** 元数据/物理状态分离 | `ProfileService.SetPackageEnabledAsync` 重命名为 `SetPackageEnabledFlagAsync`（明确"仅元数据，不部署"），旧名标 `[Obsolete]`；唯一调用点 `MainViewModel.DeployToggleAsync` 已切换 |
| **S4** 删包污染 | 新增 Core 纯函数 `PackageReferenceCounter` + `PackageDeletionPlanner`；`PackageRepository.DeletePackageAsync` 新签名 `(packageKey, allProfiles, force)` 默认拒绝删除被启用包；`ManagementCenterWindow` / `RepositoryManagerWindow` 三处调用点全部切换 |

### 🎨 字体修复

接手时发现 `UEModManager/Fonts/Inter-*.ttf` 全部 **0 字节空占位符**，`InterFont` 资源指向不存在字形，所有中文渲染走系统 fallback（"处"等字显示异常）。

修复：嵌入思源黑体 SC 三档静态字重（Regular/Medium/Bold ~49MB），SIL OFL 1.1 许可文本随包；`CyberStyles.xaml:76` 的 `InterFont` 资源 key 沿用 → 22 处 XAML 引用零修改。

## 核心对象模型速查

| 模型 | 用途 |
|------|------|
| `HostDefinition` | 游戏宿主定义（路径 / 引擎 / Mod 目录约定） |
| `Package` | 一个 MOD 包（资源 + manifest 元数据） |
| `PackageArtifact` | 包内单个文件（来源路径 / 目标相对路径 / 类型） |
| `InstanceProfile` | 一个 MOD 方案（包列表 + 启用状态 + 优先级 + 后端类型） |
| `ProfilePackageEntry` | Profile 内的包条目（IsEnabled + Priority + Kind） |
| `ConflictRecord` | 冲突记录（Type=Path/LoadOrder/ConfigKey/...，Severity） |
| `ResolvedView` | 最终视图（合并 Profile + Overwrite + Conflict 后的"应有状态"） |
| `DeploymentPlan` | 部署计划（操作列表 + 后端类型 + ProfileId） |
| `DeploymentTransaction` | 部署事务（Status / ExecutedOperations / RollbackFailures / BackupDirectory） |
| `DeploymentOperation` | 单个文件操作（Type=Add/Replace/Remove，源路径 / 目标 / 备份） |

## 核心运行流程

### 导入包

1. 用户拖拽文件到 `ImportDialog`
2. `PackageImportService.ImportSingleAsync` → `ImportFileKindClassifier.Classify` 分流（Compressed / DirectImport / Plugin）
3. 解压 → `ModFileGrouper.GroupByBaseName` 分组 → `PreviewImageSelector.Select` 选预览图
4. `PackageKindDetector` + `ArtifactTypeDetector` 识别类型
5. 写入 `ObjectStore`，更新 `PackageRepository` 索引

### 部署（启用/禁用）

1. UI 调 `MainViewModel.DeployToggleAsync(packageKey, enable)`
2. `DeploymentPlanner.CreateTogglePlanAsync` → `TogglePlanBuilder.BuildToggleOperations` 生成操作
3. `DeploymentService.ExecuteAsync(plan)`：
   - 创建事务，**立即写一次 transaction.json**（rc2 修复 S6）
   - `BackupTargetFilesAsync` 备份将被覆盖的原始文件
   - 逐个 `ExecuteOperationAsync`（调 backend）
   - 成功 → Status=Committed；失败 → `RollbackAsync`
4. 事务成功后 `MainViewModel` 调 `ProfileService.SetPackageEnabledFlagAsync` 同步元数据

### 回滚（rc2 修复 S5/S7）

1. `RollbackAsync` 按 `ExecutedOperations.Reverse()` 逆序
2. 每步 `RollbackActionPlanner.PlanRollback(op, File.Exists)` 决定动作（DeleteAdded / RestoreFromBackup / BackupMissing / NoBackupRecorded）
3. 单步失败 → 累积到 `RollbackFailures`，**不静默成功**
4. 最终：有失败 → `PartiallyRolledBack`（扫描器强制人工核查），无失败 → `RolledBack`

### 崩溃恢复

1. 启动时 `CrashRecoveryService.ScanForCrashesAsync` 加载 `Data/Backups/*/transaction.json`
2. `CrashRecoveryScanner.Scan` 分类候选项
3. UI 弹窗展示，用户决定（回滚 / 标记失败 / Dismiss）
4. `ApplyRecoveryAsync` / `DismissTransactionAsync` 执行

## 绝对不要做的事

- ❌ 在 `UEModManager.Core` 项目里加 `using System.Windows.*` —— TFM 收紧的栏杆会立即失败
- ❌ 在 Core 里做 IO（`File.*` / `Directory.*` / `HttpClient`）—— 用 Func 注入或交给主项目适配
- ❌ 直接调 `ProfileService.SetPackageEnabledAsync`（已 [Obsolete]）—— 走 `MainViewModel.DeployToggleAsync`
- ❌ 直接调 `PackageRepository.DeletePackageAsync(string)` 旧重载（已 [Obsolete]）—— 走带 `allProfiles` 的新签名
- ❌ 在事务执行中静默吞异常 —— 累积到 `RollbackFailures` 让扫描器看到
- ❌ 提交 `bin/` `obj/` `*.log` `.playwright-mcp/` `.claude/` 等（`.gitignore` 已就位）
