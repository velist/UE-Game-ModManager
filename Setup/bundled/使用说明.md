# UE Mod Manager 使用说明

> 给玩家看的简明手册。出问题翻 [`故障排查.md`](./故障排查.md)。

---

## 一图看懂

```
┌──────────────────────────────────────────────────────┐
│ 1) 装上软件   →   2) 选游戏 + 设路径                  │
│         ↓                                              │
│ 3) 把 MOD 拖进窗口   →   软件自动整理分类              │
│         ↓                                              │
│ 4) 点开关启用想用的 MOD                                │
│         ↓                                              │
│ 5) 启动游戏，进游戏看效果                              │
└──────────────────────────────────────────────────────┘
```

5 分钟就能上手。下面按窗口/按钮分块讲。

---

## 1. 第一次使用

**装好软件后第一次打开会做这几件事：**

1. 弹**登录窗口**
   - **想云同步**：注册一个账号（邮箱 + 密码），收验证码激活；以后换电脑登录，MOD 配置自动跟过来
   - **不想注册**：点"跳过"用离线模式，所有数据存本地，照样能用
2. 选**游戏**（左上角下拉框）
3. 设**游戏路径**（首次会自动弹窗，选游戏的根目录，就是 `xxx.exe` 那一层）
4. 完成，进入主界面

**支持哪些游戏？** 内置 9 款（剑星、黑神话悟空、明末无双、光与影 等）。下拉框最底部"添加自定义游戏"还能加新游戏。

## 2. 主界面

```
┌──────────────────────────────────────────────────────┐
│ [游戏▼]  [方案▼]   导入 启动  ⚙️ 头像                │
├────────┬─────────────────────────────────────────────┤
│        │ MOD 列表                                     │
│ 分类   │ ┌──────────────────────────────────────┐    │
│        │ │ ⬜ MOD 名称        预览图  优先级 开关│    │
│ ──全部 │ ├──────────────────────────────────────┤    │
│ ──人物 │ │ ...                                  │    │
│ ──服装 │ └──────────────────────────────────────┘    │
│ ──武器 │                                              │
│ +      │                                              │
└────────┴─────────────────────────────────────────────┘
```

| 区域 | 作用 |
|------|------|
| 左上"游戏" | 切游戏，每个游戏的 MOD 互不干扰 |
| 左上"方案" | 切方案（解释见 §6）|
| "导入"按钮 | 把 MOD 加进来（也可以直接拖文件进窗口）|
| "启动"按钮 | 一键开游戏 |
| ⚙️ | 设置 |
| 头像 | 账号、登录状态 |
| 左侧分类 | 给 MOD 按主题分组 |
| 中间列表 | 你导入过的所有 MOD，**用开关启用/禁用** |

## 3. 把 MOD 加进来

**最快的办法**：把 MOD 文件**直接拖到主窗口**。

支持哪些文件：

- 压缩包：`.zip` / `.rar` / `.7z`（含里面套娃的也行）
- MOD 本体：`.pak` / `.utoc` / `.ucas`
- 插件：`.dll`
- 配置：`.ini` / `.json` / `.cfg`
- 缩略图：MOD 文件夹里如果有 `preview.png` 之类的图，会自动当封面

**导入完成 ≠ 立刻生效**，得到 MOD 列表里把开关打开（见 §4）。

---

## 4. 启用 / 禁用 MOD

MOD 列表每行右边有个**开关**。

- **打开** = 把 MOD 装进游戏（程序会自动备份原文件，安全）
- **关闭** = 把 MOD 从游戏里撤出（自动还原成原样）

**其他玩法：**

- 多选后右键 → 批量启用/禁用/删除
- 拖动 MOD 调整顺序，**越靠上优先级越高**
- 列表右键 → "查看详情" / "打开包含目录"
- 左上搜索框可按名字过滤

**重要提示：**

- 启用过程中**不要强关程序**。万一真崩溃了，下次打开会自动弹"恢复"窗口（见 §10）
- 删除 MOD 前先关掉它的开关，否则程序会拦你（防止游戏目录留垃圾）

---

## 5. 分类管理

**左侧分类栏的玩法：**

- 点"+"新建分类，选个图标
- **把 MOD 拖到分类**就能归档；右键 MOD → "移动到分类"也行
- 分类可拖动改顺序、右键改名/删除
- **删分类不会删 MOD**，里面的 MOD 会落到"未分类"

**程序还会自动分类：** 导入时按 MOD 名字判断（含"hair/头发"=发型，含"weapon/武器"=武器 等），省心。

---

## 6. 方案（一组 MOD 配置的快照）

**什么场景用？**

- 想做一套"剧情通关"的 MOD 组合 + 一套"截图美化"的组合，**一键切换**
- 想把当前配置发给朋友让他一键还原

**怎么用：**

1. 主界面"方案管理"按钮 → 弹出方案管理窗口
2. 点"+"新建一个方案，起个名（比如"截图美化"）
3. 在主界面调好 MOD 启停 + 顺序 → 方案自动记住
4. 想切回普通玩法？顶部"方案"下拉框选另一个就行

**分享给朋友：**

| 类型 | 体积 | 使用场景 |
|------|------|---------|
| **配置文件**（`.lock.json`） | 几 KB | 朋友手里已有同样的 MOD，发这个让他抄你的启停顺序 |
| **整合包**（`.bundle.zip`） | 几百 MB ~ 几 GB | 朋友啥都没有，发这个他双击导入就能用 |

方案管理窗口里都有"导出/导入"按钮。

**小坑（v2.0-rc2 已知）：** 切方案后**手动点一下"应用方案"**才会真正部署，不点的话游戏里看到的还是上一个方案。下个版本会修。

---

## 7. 部署预览 + 冲突分析

**部署预览**（"部署预览"按钮）：

- 启用前先看清楚程序要给你游戏目录加哪些文件、覆盖哪些文件
- 觉得对了再确认

**冲突分析**（"冲突分析"按钮）：

- 如果两个 MOD 都想改同一个文件，会列出来
- 你可以**手动选择哪个 MOD 赢**（叫"override"，按游戏单独保存）
- 不选的话默认按优先级（列表中靠上的赢）

**冲突类型说人话：**

- "加载顺序冲突" = 两个 MOD 想改同一个游戏文件
- "配置键冲突" = 两个 MOD 想改同一个设置（程序会试着自动合并）

---

## 8. 管理中心（仓库 + 维护工具）

主界面"管理中心"按钮打开。

**4 个标签页：**

- **仓库**：看你导入过的所有 MOD 包，可单独删
- **覆盖文件**：被 MOD 改掉的原始文件备份，可一键还原
- **健康检查**：开机自检的最近报告（路径对不对、磁盘有没有满 等）
- **导出诊断包**：底部按钮，**出问题时点这个**，自动打包日志发开发者排查（敏感信息已自动遮掉）

**两个清理工具：**

- "清理未使用的文件" — 删除不在任何方案里的孤儿 MOD
- "合并相同文件" — 删除内容一模一样的重复 MOD

---

## 9. 启动游戏

**两种方式：**

- 顶部"启动"按钮 — 直接开游戏
- "启动中心"窗口 — 启动前可选"先做一次冲突检查再开"，更稳

**注意：** 游戏开起来之后**别在管理器里改 MOD**，文件被游戏占用，会出错。先关游戏再改。

---

## 10. 出问题时的"恢复"窗口

如果上次启用/禁用过程中程序崩了 / 强关了，**下次打开会自动弹这个窗口**，列出"上次没干完的活"。

**怎么选？**

| 它说啥 | 你点啥 |
|--------|-------|
| "上次执行中崩溃" | **点"回滚"**（撤销那次操作） |
| "之前回滚也失败了" | **点"人工核查"**，看清楚哪些文件没还原好，必要时找开发者 |
| "事务日志写盘失败" | **去看一眼游戏目录是否正常**，正常就忽略，不正常就重新启用一次 |
| "已失败但没清理" | **点"标记失败"**（清掉残留）|
| "创建后没启动" | **点"标记失败"** |

**不想再被提醒？** 点"忽略本次"，写进档案不再骚扰。后悔了到管理中心 → "重置已忽略事务"。

---

## 11. 设置

⚙️ 图标打开。常用项：

- **语言** — 中 / 英
- **路径** — 改 MOD 部署目录、备份目录
- **部署方式**：
  - **文件复制** — 默认，最稳
  - **节省空间模式（硬链接）** — 不占双倍空间；MOD 必须和游戏在**同一个磁盘**，跨磁盘时自动改用复制
- **关于** — 版本号、检查更新、查看许可

**改部署方式后**：建议先把所有 MOD 都禁用 → 切方式 → 再重新启用一次（已经部署的不会自动迁移到新方式）。

## 12. 老版本升级到 v2.0

**最简单的办法：双击 `一键迁移老版本数据.bat`**（和管理器装在一起的）。

它会自动：

1. 把你 `%APPDATA%\UEModManager\` 整个目录**完整备份**到 `UEModManager-backup-时间戳\`（双重保险）
2. 启动管理器
3. 管理器检测到老数据后会弹"迁移向导"，5 步全自动（扫描 → 整理 → 迁移 → 校验 → 完成），点"下一步"到底就行

**完事一周后觉得没问题**，可以手动删 `UEModManager-backup-xxx` 目录释放空间。

**手动迁移**（不想用脚本）：

1. 资源管理器地址栏粘贴 `%APPDATA%` 回车
2. 找到 `UEModManager` 文件夹，整个**复制一份**（粘贴出来加日期改个名）
3. 启动管理器，按弹出的迁移向导走完即可
4. 出问题就把备份的文件夹改回 `UEModManager`

## 13. 文件都放哪了

| 内容 | 位置 | 怎么打开 |
|------|------|---------|
| 软件本体 | 安装目录 | 桌面快捷方式右键"打开文件位置" |
| 你的存档/账号/分类 | `%APPDATA%\UEModManager\` | 资源管理器地址栏粘贴回车 |
| MOD 包仓库 | 软件目录 `Data\Repository\` | — |
| 备份文件 | 软件目录 `Data\Backups\` | — |
| 日志（出问题时给开发者） | 软件目录 `console.log` | — |

**搬家 / 重装系统？** 复制 `%APPDATA%\UEModManager\` 整个目录到新机器同位置，重装软件即可。
